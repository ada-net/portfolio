How to SEO-friendly use H1-H6 for design-only purposes

Once you will be ready to work as a frontend developer, you might get in touch also with search engine optimizers (SEOs) whose work it is to check if the site you are developing is made in the best possible way to rank on search engine result pages. One of the most common SEO requests will sound like that: "Heading elements should not be used for styling purposes.".

Find out what that means and what to do in order to make happy your client, Google and the SEO specialist!


Search engines need to understand a page's content

A search engine does not only index pages, it also needs to try to understand what a page is talking about.



If you want to know more about how search engines work, take a look at this guide written by Google itself.


Only if Google figures out that your shop is selling flowerpots and that this shop is well-organized and able to satisfy costomer's needs, it will consider showing it to people searching for flowerpots.

Not always understanding a page's content is easy for Google and Co.

Next to other elements contributing to a good interpretation of the content (e.g. semantic tags), headings and the heading structure play important roles, too.

How headings can help search engines to understand a page's content

The best way to imagine which heading could be useful and which not, is to imagine the page and it's elements you're developing as if they should be used to compone a book's index. And exactly that is what can help search engines to understand the topic of a page and it's subtopics.

Let's imagine we have to design a blog article about next month's events in Rome for our client.
Thinking about this content as a book's index helps choosing a useful heading structure.

An ideal structure could be:

  • H1: Events in Rome for June 2021
  • H2: Concerts
  • H3: Depeche Mode on June 2nd
  • H3: Andrea Bocelli on June 11th
  • ...
  • H2: Guided Tours
  • H3: Antique Rome tour on June 9th
  • H3: Medieval Rome tour on June 15th
  • ...

Wow, that's really a helpful index that helps search engines understand what the page is talking about!

Unfortunately, these perfect signals often don't come alone. Your client for example might want to insert some CTA phrases between the lines of the main content of this page. And these CTAs obviously should be immediately visible and, for design reasons, look like a heading. At this point you should ask yourself if the CTA should become a chapter in our book's imaginary index.

The answer is: no, it wouldn't make sense.
So, in some situations you might want a heading-like style without using a real HTML heading.

Create heading-like styles

So, the solution for creating text elements who look like a heading but aren't, is simple and costs only a few minutes:

Why not create heading-like css classes?

Maybe there is a more intelligent solution, but I have solved like this:

  1. Write your usual css styles for h1 - h6 as you always do
  2. Once these styles are approved by your client, create dedicated classes for your heading-like styles. Example:
    • .h1-like {put the approved h1 properties here}
    • .h2-like {put the approved h1 properties here}
    • ... and so on ...
  3. Apply your heading-like classes to the elements who have to look like a heading but don't have semantic value.
